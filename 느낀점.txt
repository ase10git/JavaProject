기획 동기 : 약 한달동안 배우고 익힌 java의 내용들을 최대한 활용하여 project를 진행하기 위해 키오스크 라는 주제를 선정하게 됨 

개발 목표 : 
휴게소에서 먹고자 하는 음식을 선택하고 계산하는데 사용되는 키오스크 만들기

소프트웨어 시나리오 : 
1. 메인 화면에서 매장 이용 방법을 선택한다 (포장하기, 매장식사)
2. 분식, 일식, 한식, 양식 메뉴 카테고리마다 음식 메뉴들이 나타나고 원하는 갯수만큼 수량 선택
3. 선택한 음식과 수에 맞게 장바구니에 담기게 되고 주문하고자 하는 음식들의 값이 보여진다.
3-1. 만약 잘못 선택되었거나, 변심이 생겨 장바구니에 담긴 음식을 변경하고자 할 경우 장바구니에 담긴 해당 메뉴를 선택하여 제거 할 수 있다
3-2. 장바구니에 아무것도 없을 경우 결제 버튼을 눌러도 결제 창으로 넘어가지 않는다..
4. 결제 버튼을 클릭하여 결제창으로 넘어간다.
5. 결제해야할 총 금액이 화면에 표시되고 결제 방법을 선택한다.
6. 선택된 결제 방법으로 결제가 이루어지고 결제가 완료되면 10초 뒤 처음 화면으로 돌아간다.


menu패널

초기에는 메뉴 이미지를 gridLayout으로 각 카테고리별 패널에 꽉 차게 하여 크게 보여주려 하였으나 그렇게 되면 메뉴 이름,가격에 대한 정보를 보여줄 공간이 없어서 이를 해결하고자 작은 크기의 버튼들을 일정한 간격에 맞게 배치해 주고 그 버튼에 메뉴 이미지를 넣는 방식을 채택하였습니다.

카테고리 별로 패널을 설정해 주었고
각 카테고리별 메뉴와 가격을 2차원 배열로 저장하였고, 각각 카테고리별 메뉴에 대한 정보들(음식 이름,사진,가격 등)은 반복문을 통하여 패널에 입력해 주었습니다.

음식 이미지는 사진의 크기를 조정하여 버튼에 넣는 방식으로 하였고 이미지를 불러오는 코드 부분 또한 반복문을 통하여 (주소)+음식 이름+확장자명 으로 설정하여 조금 더 간단한 코드로 이미지를 불러올 수 있었습니다.

각각의 버튼동작들은 각 버튼의 원하는 동작에 맞게ActionListener으로 코드 작성

느낀점 : 혼자 코드를 작성하는 것이 아닌 협업을 통하여 코드를 작성하려니 어디서부터 어떻게 시작을 해야 각자 수정을 하고 코드를 합칠 수 있을지에 대해 초반부분에 어려움을 느낌. 
코드가 길어지다보니 수정하고자 하는 부분의 코드를 찾는데 어려움을 느낌.
